Personal Finance Dashboard

Final Web Application Project Proposal

1. Overview

Problem: People struggle to manage their finances due to fragmented information across multiple platforms. Tracking investments, monitoring spending, and staying informed requires juggling multiple apps.

Solution: A comprehensive web dashboard that consolidates portfolio tracking, spending analysis, and financial news into one intuitive interface using vanilla JavaScript.

Motivation: Financial literacy impacts quality of life. This project empowers users to take control of their finances while demonstrating proficiency in vanilla JavaScript, API integration, data visualization, and responsive design.

2. Target Audience

  • Young Professionals (25-35): Tech-savvy individuals beginning their investment journey who want to track portfolios and manage spending
  • Retail Investors: Active traders wanting real-time portfolio updates and market insights
  • Budget-Conscious Individuals: Anyone looking to understand and optimize spending patterns

User Needs: Real-time portfolio valuation, historical performance tracking, spending analysis, financial news, goal tracking, and mobile-friendly access across devices.

3. Major Functions

Function 1: Portfolio Management

Users can add, edit, and delete stock holdings. Each holding includes stock symbol, quantity, purchase price, and date. Data stored in localStorage with search, filter, and sort capabilities. Includes bulk CSV import/export functionality.

Function 2: Real-Time Portfolio Valuation

Fetches current stock prices from Alpha Vantage API and calculates total portfolio value in real-time. Displays current value, total gain/loss (dollars and percentage), daily change, and color-coded performance indicators (green for gains, red for losses).

Function 3: Interactive Performance Charts

Visual representation of portfolio performance using Chart.js or Canvas API. Line graphs show value trends over time, pie charts display asset allocation. Multiple time period views (1 day, 1 week, 1 month, 3 months, 1 year, all time) with interactive tooltips and zoom capabilities.

Function 4: Transaction History Tracking

Complete record of all buy/sell transactions with date, symbol, type, quantity, price, and total value. Filter by date range or stock symbol. Automatically calculates cost basis and realized gains/losses. Export transaction history to CSV.

Function 5: Spending Analysis and Categorization

Log expenses and categorize them (Food, Transport, Entertainment, Bills, Shopping, Healthcare, etc.). Visual breakdowns with bar and pie charts. Set monthly budgets per category with visual indicators and warnings when approaching 90% of limits.

Function 6: Financial Goals Management

Set and track multiple financial goals (savings targets, emergency fund, vacation fund, etc.). Each goal includes target amount, current progress, deadline, and notes. Visual progress bars show completion percentage. System calculates required monthly savings to meet deadlines.

Function 7: Personalized Financial News Feed

Integration with News API to display relevant financial news articles. Filters news based on stocks in user's portfolio and general market news. Search functionality, date filtering, and article bookmarking. Each article shows headline, summary, source, date, and link.

Function 8: Currency Conversion and Multi-Currency Support

Real-time currency conversion using ExchangeRate-API. View portfolio value in different currencies (USD, EUR, GBP, JPY, CAD, AUD, CHF, CNY). Historical exchange rates for accurate performance tracking. Built-in currency converter tool.

Function 9: Dashboard Overview and Analytics

Comprehensive home page aggregating all key metrics: portfolio summary cards, top gainers/losers, recent transactions, spending summary, goal progress, and latest news. Responsive CSS Grid layout adapts to mobile, tablet, and desktop. Customizable widgets.

Function 10: Data Management and Settings

Settings panel for managing preferences and data. Export all data (portfolio, transactions, expenses, goals) as JSON or CSV. Import data from files. Clear/reset all data with confirmation. Customize display preferences: currency, refresh intervals, theme (dark/light), chart colors.

5. External Data

API 1: Alpha Vantage (Stock Market Data)

Purpose: Real-time and historical stock prices

Endpoints:

  • GLOBAL_QUOTE - Current stock price and daily performance
  • TIME_SERIES_DAILY - Historical daily prices for charts
  • SYMBOL_SEARCH - Search and validate stock symbols

Data Needed: Current price, daily change, volume, historical prices

Rate Limit: Free tier 25 requests/day (will use caching)

API 2: News API (Financial News)

Purpose: Relevant financial and market news

Endpoints:

  • Search news by keyword (stock symbols)
  • Top business/finance headlines
  • Filter by date and source

Data Needed: Headline, summary, date, source, URL, thumbnail

Rate Limit: 100 requests/day (will cache articles)

API 3: ExchangeRate-API (Currency Conversion)

Purpose: Real-time currency exchange rates

Endpoints:

  • Latest exchange rates
  • Historical rates
  • Currency conversion

Data Needed: Exchange rates for USD, EUR, GBP, JPY, CAD, AUD

Rate Limit: 1,500 requests/month (daily cache updates)

Data Storage (localStorage)

  • Portfolio Holdings: Symbol, quantity, avg cost, purchase date
  • Transactions: Date, type, symbol, quantity, price, notes
  • Expenses: Date, category, amount, description, merchant
  • Goals: Title, target amount, current amount, deadline
  • Settings: Currency, theme, refresh interval
  • Cache: Stock prices, exchange rates, news (with timestamps)

6. Module List

1. App Module (app.js)

Main entry point, initialization, routing, global state management

2. Storage Module (storage.js)

localStorage CRUD operations, data validation, import/export, clear data

3. API Manager (api-manager.js)

Centralized API requests, caching, rate limiting, error handling

4. Stock Service (stock-service.js)

Fetch stock prices, historical data, search symbols, calculate metrics

5. Portfolio Manager (portfolio-manager.js)

Add/edit/delete holdings, calculate statistics, asset allocation

6. Transaction Manager (transaction-manager.js)

Record transactions, update holdings, calculate gains/losses

7. Spending Tracker (spending-tracker.js)

Add/edit expenses, categorize, budget tracking, spending reports

8. Goals Manager (goals-manager.js)

Create/track goals, calculate savings rate, progress monitoring

9. News Service (news-service.js)

Fetch news, filter by portfolio, search, cache articles

10. Currency Service (currency-service.js)

Fetch exchange rates, convert currencies, format displays

11. Chart Manager (chart-manager.js)

Create charts (line, pie, bar), handle interactions, responsive sizing

12. UI Manager (ui-manager.js)

Render views, handle navigation, update elements, modals, notifications

13. Form Handler (form-handler.js)

Validate forms, handle submissions, display errors, pre-populate data

Date/number formatting, generate IDs, debounce, DOM helpers

15. Router (router.js)

Client-side routing, URL hash changes, load views, history management

10. Challenges

1. API Rate Limiting

Challenge: Alpha Vantage free tier only allows 25 requests per day, which is very restrictive for a portfolio tracker.

Solution: Implement intelligent caching strategy with localStorage. Cache stock prices with timestamps and only refresh when data is older than 5 minutes during market hours. Use batch requests when possible. Implement manual refresh button to give users control.

2. Real-Time Data Without WebSockets

Challenge: Providing "real-time" updates using vanilla JavaScript without WebSocket libraries or frameworks.

Solution: Use setInterval for polling API at reasonable intervals (5-10 minutes). Implement smart refresh that only updates during market hours (9:30 AM - 4:00 PM EST). Provide manual refresh option for instant updates when needed.

3. Complex State Management Without Framework

Challenge: Managing application state across multiple modules without React/Vue state management.

Solution: Create centralized state management using ES modules with a pub/sub pattern. Implement custom event system for component communication. Use localStorage as single source of truth and trigger UI updates when localStorage changes.

4. Chart Creation Without Heavy Libraries

Challenge: Creating interactive, responsive charts using only Chart.js (allowed CSS library).

Solution: Learn Chart.js thoroughly and create reusable chart configurations. Optimize chart rendering for performance. Consider Canvas API fallback for simple visualizations if needed.

5. Data Persistence and Synchronization

Challenge: Keeping cached API data synchronized with localStorage and ensuring data integrity.

Solution: Implement timestamp-based cache validation. Create data migration system for localStorage schema updates. Add data export/import functionality as backup mechanism.

6. Mobile Responsiveness for Data-Heavy UI

Challenge: Displaying complex tables, charts, and multiple data points on mobile screens.

Solution: Design mobile-first with priority on essential information. Use CSS Grid and Flexbox for flexible layouts. Implement collapsible sections and tabs for mobile. Optimize tables to card layouts on small screens.

7. Performance with Large Datasets

Challenge: Maintaining performance when users have large portfolios or transaction histories.

Solution: Implement pagination for large lists. Use virtual scrolling for long tables. Optimize DOM manipulation by batching updates. Lazy load charts and heavy components.

8. Cross-Browser Compatibility

Challenge: Ensuring consistent functionality across all modern browsers without build tools or polyfills.

Solution: Use well-supported ES6+ features only. Test extensively on Chrome, Firefox, Safari, and Edge. Avoid experimental CSS features. Use feature detection for browser-specific functionality.
